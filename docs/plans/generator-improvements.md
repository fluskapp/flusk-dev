# Generator Improvements Plan

## Goal
All generated files must be under 100 lines. All generators must produce
split output when the single-file result would exceed the limit.

## Current State (Feb 23, 2026)
- 34 files over 100 lines (4 @generated, 30 hand-written forge code)
- Generated oversized files: repository (146), parse-readable-span (206),
  app.ts (143), hooks (134), types/index.ts (133)
- Hand-written forge files being split by sub-agent

## Issues to Fix

### 1. Trait Composer (crud-builders.ts, crud-route-builders.ts)
The trait composer generates monolithic repository + route files.
It should delegate to multi-file generators instead.

**Fix:** Make trait composer output reference the split files from
sqlite-repo recipe instead of generating its own monolithic version.
Remove duplicate: `llm-call.repository.ts` (monolithic) vs `llm-call/` (split).

### 2. Barrel Generator (types/index.ts)
The barrel generator puts all exports in one file.

**Fix:** When >20 exports, generate sub-barrels by category:
- `types/entities.ts` (entity types)
- `types/operations.ts` (insert/update/query)
- `types/index.ts` (re-exports sub-barrels)

### 3. Pipeline Generator
Fixed fn: steps and import paths. Still needs:
- Split output for pipelines >100 lines (unlikely but possible)
- Type-safe fn: calls (currently uses any)

### 4. Route Generator
The trait composer route output doesn't match codebase patterns.
`llm-call.routes.ts` is intentionally kept as a stub.

**Fix:** Route trait should generate split route files (like llm-calls-routes/)
or skip if split directory already exists.

### 5. OTel parse-readable-span (206 lines)
Generated by otel-hook recipe. Needs split into:
- `parse-readable-span.ts` (main function, <50 lines)
- `parse-genai-attributes.ts` (GenAI attribute parsing)
- `sanitize-span-data.ts` (security sanitization)

### 6. App.ts (143 lines)
Generated by middleware recipe. Plugin registration grows with entities.

**Fix:** Generate `register-plugins.ts` helper, keep app.ts under 80 lines.

## Design Principles
1. When output would exceed 80 lines, split into sub-files
2. Main file becomes an orchestrator (<50 lines) importing helpers
3. Sub-files follow naming convention: `{entity}/{concern}.ts`
4. Barrel exports maintain backward compatibility
5. Each generator should have a `splitThreshold` config (default: 80)
