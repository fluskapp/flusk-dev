# Flusk CLI Usage Guide

## Quick Start

```bash
# Generate from a single entity
pnpm flusk g llm-call.entity.ts

# Generate from all entities
pnpm flusk g --all

# Generate types only
pnpm flusk g pattern.entity.ts --types-only

# Run migrations
pnpm flusk migrate
```

## Commands

### `flusk g [entity-file]`

Generate all code layers from entity schema.

**Arguments:**
- `entity-file` - Entity filename (e.g., `llm-call.entity.ts`)

**Options:**
- `--types-only` - Generate only types package (skip resources, business-logic, execution)
- `--all` - Generate from all entities in `packages/entities/src/`

**Examples:**

```bash
# Single entity - all layers
pnpm flusk g llm-call.entity.ts

# Single entity - types only
pnpm flusk g llm-call.entity.ts --types-only

# All entities
pnpm flusk g --all
```

### `flusk migrate`

Run all SQL migrations in order.

**Requirements:**
- `DATABASE_URL` environment variable must be set

**Example:**

```bash
DATABASE_URL=postgresql://... pnpm flusk migrate
```

## Generated Files

For entity `llm-call.entity.ts`, generates:

### Types (`packages/types/`)
- `llm-call.types.ts`
  - `LLMCallEntity` type
  - `LLMCallInsert` type (excludes id, timestamps)
  - `LLMCallUpdate` type (partial with id required)
  - `LLMCallQuery` type (all optional)
  - JSON schemas for validation

### Resources (`packages/resources/`)
- `repositories/llm-call.repository.ts`
  - `create()` - Insert record
  - `findById()` - Get by UUID
  - `update()` - Update record
  - `closePool()` - Cleanup
- `migrations/NNN_llm_calls.sql`
  - Table creation
  - Indexes
  - Triggers

### Business Logic (`packages/business-logic/`)
- `llm-call/validate-llm-call.function.ts` - Validation stub
- `llm-call/index.ts` - Exports

### Execution (`packages/execution/`)
- `routes/llm-call.routes.ts`
  - POST /llm-calls
  - GET /llm-calls/:id
- `plugins/llm-call.plugin.ts` - Fastify plugin
- `hooks/llm-call.hooks.ts` - Request hooks

## File Headers

All generated files include:

```typescript
/**
 * @generated by @flusk/cli from entity-name.entity.ts
 * DO NOT EDIT - regenerate using: flusk g entity-name.entity.ts
 */
```

## Workflow

1. Create/modify entity schema in `packages/entities/src/`
2. Run `pnpm flusk g entity-name.entity.ts`
3. Review generated files
4. Customize business logic and hooks (NOT generated files)
5. Run `pnpm flusk migrate` to update database

## Architecture

The CLI follows Flusk's composable monolith architecture:

- **Schema-first**: TypeBox → JSON Schema → TypeScript
- **Pure functions**: Business logic has no I/O
- **I/O isolation**: Resources handle all database/API calls
- **Plugin-based**: Each entity gets its own Fastify plugin
- **100-line limit**: All files split to stay under 100 lines

```bash
# Test

```

## Error Handling

The CLI will exit with error if:
- Entity file not found
- Invalid entity schema
- File write permission issues
- DATABASE_URL not set (for migrations)
